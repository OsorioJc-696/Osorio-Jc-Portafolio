Semana 1

Temas

  • Tecnoloígas web básicas
  • Lengiaje web (HTML, CSS, JavaScript)
  • Gráficos en la web
  • Funcionamiento de la web: DNS
  • Protolo TPC/IP
  • Roles en el desarrollo de aplicaciones para internet
  • Desarrollador Frontend
  • Desarrollador Backend
  • Desarrollador Fullstack
  • Herramienta de desarrollo - Visual Studio Code (VSC)

Definiciones

1. Tecnologías web básicas: Conjunto fundamental de herramientas para desarrollar sitios y aplicaciones web, incluyendo HTML (estructura), CSS (diseño) y JavaScript (interactividad).
2. Lenguaje web (HTML, CSS, JavaScript): Elementos esenciales para crear páginas web: HTML define la estructura, CSS controla el diseño y JavaScript permite la interacción.
3. Gráficos en la web: Elementos visuales como imágenes y videos utilizados en sitios web para comunicar información y mejorar la experiencia del usuario.
4. Funcionamiento de la web - DNS (Sistema de Nombres de Dominio): Sistema que traduce nombres de dominio legibles por humanos en direcciones IP, facilitando la localización de recursos en Internet.
5. Protocolo TCP/IP: Conjunto de reglas para la comunicación en Internet; TCP garantiza la entrega de datos y IP dirige el tráfico en la red.
6. Roles en el desarrollo de aplicaciones para Internet: Diferentes responsabilidades en la creación de aplicaciones web, como desarrollador frontend (interfaz de usuario), backend (lógica del servidor) y fullstack (ambos).
7. Desarrollador Frontend: Crea la parte visible de una aplicación web, utilizando HTML, CSS y JavaScript para diseñar una experiencia atractiva para el usuario.
8. Desarrollador Backend: Maneja la lógica y la gestión de datos en el servidor de una aplicación web, asegurando su funcionamiento eficiente.
9. Desarrollador Fullstack: Combina habilidades de desarrollo frontend y backend, lo que les permite trabajar en todas las capas de una aplicación web.
10. Herramienta de desarrollo - Visual Studio Code (VSC): Un entorno de desarrollo altamente personalizable y ampliamente utilizado para escribir, depurar y gestionar código en diversos lenguajes, incluyendo tecnologías web.

Procedimiento

° Explorar los fundamentos de las tecnologías web esenciales, abarcando la creación de contenido en línea a través de HTML, el diseño web mediante CSS y la implementación de interacción con JavaScript.
° Investigar y comprender el funcionamiento del Sistema de Nombres de Dominio (DNS) y su capacidad para convertir nombres de dominio en direcciones IP, permitiendo así la localización de recursos en la red.
° Estudiar el rol del Protocolo TCP/IP en la infraestructura de comunicación de Internet, junto con el Protocolo HTTP utilizado para la transferencia de datos en la web.
° Analizar las diversas funciones desempeñadas en el desarrollo de aplicaciones web, incluyendo el desarrollo del frontend, que se centra en la interfaz de usuario, el backend, que gestiona la lógica del servidor, y los desarrolladores fullstack, que tienen conocimientos en ambas áreas.
° Adentrarse en Visual Studio Code como una herramienta de desarrollo, explorando sus elementos de interfaz de usuario, las extensiones disponibles y los atajos de teclado para optimizar la escritura y gestión de código.

Resultado

° Adquisición de destrezas en la aplicación de tecnologías web fundamentales como HTML, CSS y JavaScript.
° Comprensión de cómo el Sistema de Nombres de Dominio (DNS) habilita la navegación en línea al convertir nombres de dominio en direcciones IP.
° Dominio del protocolo TCP/IP y su relevancia en la comunicación en la red, junto con el protocolo HTTP en la transmisión de datos.
° Reconocimiento de roles específicos en el proceso de desarrollo de aplicaciones web, que incluyen frontend, backend y fullstack.
° Aprovechamiento efectivo de Visual Studio Code como una herramienta de desarrollo, prestando atención a componentes de la interfaz, extensiones y métodos abreviados de teclado.

Definiciones

1. Lenguaje Marcado de Hipertexto: Un lenguaje utilizado para estructurar y formatear el contenido en la web, como HTML (HyperText Markup Language), que define la organización de elementos en una página web.
2. Lenguaje Marcado Extensible (XML): Un estándar que permite la creación de etiquetas personalizadas para describir datos y documentos, proporcionando flexibilidad en la representación de información estructurada.
3. Open Web Platform: Un conjunto de estándares abiertos, tecnologías y directrices que respaldan el desarrollo de aplicaciones web, promoviendo la interoperabilidad y la accesibilidad.
4. Árbol DOM (Document Object Model): Una representación jerárquica de una página web que permite a los programadores acceder y manipular los elementos de una página web utilizando JavaScript u otros lenguajes de programación.
5. Renderizado de Árbol: El proceso de construcción y visualización de la página web a partir del árbol DOM, donde los elementos se muestran en el navegador según su estructura y estilos definidos.
6. Ciclo de Vida de una Página Web: El conjunto de fases por las que pasa una página web desde que se solicita hasta que se muestra en el navegador, incluyendo la solicitud, el procesamiento en el servidor y el renderizado en el cliente.
7. Optimización en motores de búsqueda (SEO): Un conjunto de estrategias y técnicas utilizadas para mejorar la visibilidad de un sitio web en los resultados de búsqueda, aumentando su tráfico orgánico y su posición en los motores de búsqueda como Google.
8. Compatibilidad entre navegadores: Asegurar que un sitio web o una aplicación web se visualice y funcione de manera consistente en diferentes navegadores web, como Chrome, Firefox, Edge, y Safari, evitando problemas de incompatibilidad.
9. Hojas de Estilos CSS3: CSS3 (Cascading Style Sheets nivel 3) es una versión del lenguaje de hojas de estilo que se utiliza para diseñar y dar estilo a páginas web. CSS3 introduce características avanzadas, como transiciones, animaciones y sombras, para mejorar la presentación de los elementos web.

Procedimiento

° Elaboración de páginas HTML y XML con etiquetas adecuadas.
° Desarrollo de una jerarquía de elementos HTML y XML.
° Manipulación del Modelo de Objetos del Documento (DOM) para interactuar y modificar dinámicamente el contenido web.
° Aplicación de estrategias de optimización para motores de búsqueda (SEO), como la inclusión de metatags y palabras clave.
° Realización de pruebas exhaustivas en varios navegadores para asegurar la compatibilidad.
° Utilización de hojas de estilo CSS3 para implementar diseños, incluyendo flexbox y diseño adaptable (responsive design).
° Creación de diseños web adaptables que se ajusten a distintos dispositivos.

Resultado

° Creación exitosa de sitios web utilizando HTML y XML, adhiriéndose a estándares web.
° Utilización efectiva del Modelo de Objeto de Documento (DOM) para acceder y manipular el contenido en línea.
° Implementación de tácticas de optimización para mejorar la visibilidad en motores de búsqueda.
° Asegurando la interoperabilidad entre navegadores para brindar una experiencia uniforme.
° Diseño de plataformas en línea atractivas y funcionales con hojas de estilo CSS3.
° Desarrollo de sitios web adaptables que se ajustan a diversas dimensiones de pantalla y tipos de dispositivos.

Semana 3

Temas:

  • Librerías CSS Front-end
  • Diseño de Tipografpia
  • Layout y Components
  • Flexbox y diseño Responsivo

Definiciones

1. Librerías CSS Bootstrap y Tailwind: Son herramientas de desarrollo web que ofrecen conjuntos de estilos predefinidos y componentes para simplificar la creación de páginas web Front-End. Bootstrap es conocido por su conjunto de componentes y diseño, mientras que Tailwind se enfoca en clases de utilidad para personalizar estilos.
2. Diseño de Tipografía: Se refiere a la elección y configuración de fuentes, tamaños de texto, espaciado de línea, colores de fondo y bordes para garantizar una presentación legible y atractiva del texto en una página web.
3. Layout y Components: Implica la organización de elementos en un diseño web, incluyendo la estructura de contenedores, la disposición de columnas, y la creación y personalización de componentes de interfaz de usuario.
4. Flexbox y Diseño Responsivo: Flexbox es un modelo de diseño que permite la creación de diseños flexibles y responsivos al proporcionar un control preciso sobre la disposición y el espacio entre elementos en una página web.
5. Componentes de Interfaz de Usuario: Son elementos interactivos y visuales, como botones, formularios, avatares y otros, que los usuarios pueden ver y utilizar en una página web para realizar acciones o acceder a información. Estos elementos mejoran la experiencia del usuario y permiten la interacción con la página web.

Procedimiento

° Elección de Biblioteca de Estilos CSS: Seleccionar entre Bootstrap y Tailwind de acuerdo a los requerimientos del proyecto y el estilo de diseño que se busca.
° Diseño de Tipografía: Configurar las fuentes, tamaños de texto, espaciado entre líneas y otros atributos tipográficos utilizando las clases proporcionadas por la biblioteca seleccionada.
° Elaboración del Diseño de la Página: Crear la estructura de la página utilizando contenedores, columnas y elementos de diseño fluido y adaptable.
° Aplicación de Flexbox: Emplear Flexbox para organizar y posicionar los elementos de forma flexible, permitiendo adaptar el diseño a distintos tamaños de pantalla.
° Personalización de Elementos de Interfaz: Desarrollar y personalizar componentes de la interfaz de usuario, como botones, formularios y otros elementos, para que se ajusten al diseño deseado.

Resultado

° Elaboración de atractivas y funcionales interfaces Front-End para sitios web mediante el aprovechamiento de las librerías CSS Bootstrap y Tailwind.
° Incorporación de tipografía con legibilidad y estilo, realzando la presentación del contenido en las páginas web.
° Creación de diseños adaptables y flexibles que se ajustan sin problemas a diversas pantallas y dispositivos.
° Uso efectivo de Flexbox para la organización de elementos y la creación de diseños versátiles.
° Desarrollo y personalización de elementos de la interfaz de usuario con el fin de proporcionar una experiencia de usuario coherente y placentera en el diseño web.
° Destacado desempeño en la Práctica Calificada 01, que abarcó la creación de un sitio web con diseño flexbox, diseño líquido y diseño adaptable, demostrando destreza en la elaboración de diseños web avanzados.

Temas:

  • Desarrolo Front-End con JavaScript

  • Estructuras de Control y Bucles
  • Funciones en Javascript: Sintaxis, Invocación y Tipos
  • Funciones Anónimas, Función Objeto y Funciones Flecha
  • Funciones Auto Invocadas y Funciones Closures
  • Clases Number, String y Math en Javascript
  • Trabajo con Arrays Unidimensionales y Multidimensionales
  • Utilización de Objetos Map en Javascript

Definiciones

1. Desarrollo Front-End con Javascript: Este enfoque implica la creación de la interfaz de usuario y la lógica de una página web utilizando el lenguaje de programación Javascript. Los desarrolladores Front-End utilizan Javascript para mejorar la experiencia del usuario en el navegador web.
2. Motores Javascript, Variables y Operadores:** Los motores Javascript, como el motor V8, interpretan y ejecutan código Javascript en los navegadores. Las variables son contenedores que almacenan datos, y los operadores son símbolos utilizados para realizar operaciones en estos datos, como sumas o comparaciones.
3. Tipos de Datos: En Javascript, los tipos de datos incluyen números (para valores numéricos), strings (para texto), objetos (para estructuras de datos complejas), booleanos (para valores verdaderos o falsos) y otros valores, que permiten al programador trabajar con diferentes tipos de información.
4. Estructuras de Control y Bucles: Estas son instrucciones que permiten tomar decisiones (como if/else) y repetir tareas (mediante bucles como for o while) en función de condiciones específicas en el código Javascript.
5. Funciones en Javascript: Son bloques de código reutilizables que realizan tareas específicas. Se definen utilizando la palabra clave "function" y pueden tomar argumentos y devolver resultados.
6. Funciones Anónimas, Función Objeto y Funciones Flecha: Estas son diferentes formas de definir y utilizar funciones en Javascript. Las funciones anónimas no tienen un nombre definido, las funciones objeto son métodos de un objeto y las funciones flecha son una forma concisa de definir funciones.
7. Funciones Auto Invocadas y Funciones Closures: Las funciones auto invocadas se ejecutan inmediatamente después de su definición, y los closures son funciones que pueden acceder y modificar variables fuera de su propio ámbito, lo que proporciona control sobre la visibilidad de variables.
8. Clases Number, String y Math: Estas son clases integradas en Javascript que proporcionan funcionalidades avanzadas para números y cadenas, así como operaciones matemáticas. La clase Number y la clase String contienen métodos para trabajar con números y cadenas, mientras que la clase Math proporciona funciones matemáticas avanzadas.
9. Arrays Unidimensionales y Multidimensionales: Los arrays son estructuras de datos que almacenan colecciones de elementos, ya sea en una sola dimensión (unidimensionales) o en múltiples niveles (multidimensionales) organizados en forma de matriz.
10. Objetos Map: Un objeto Map es una estructura de datos en Javascript que asocia claves con valores, permitiendo un acceso eficiente a los datos. A diferencia de los objetos regulares, los Map pueden utilizar cualquier tipo de dato como clave y mantienen el orden de inserción de elementos.

Procedimiento

° Crear una página web de Front-End: Diseñar la interfaz de usuario y escribir el código Javascript para interactuar con ella.
° Utilizar intérpretes de Javascript: Ejecutar código Javascript en intérpretes como V8 para visualizar resultados en un navegador.
° Manipular variables y operadores: Declarar variables, asignar valores y efectuar operaciones aritméticas y lógicas.
° Implementar estructuras de control y bucles: Emplear instrucciones condicionales como "if" y bucles como "for" para gestionar el flujo de ejecución.
° Generar funciones: Establecer funciones con la sintaxis adecuada y llamarlas para realizar tareas específicas.
° Explorar funciones avanzadas: Utilizar funciones anónimas, objetos de función, funciones flecha, autoejecutables y cierres para abordar situaciones complejas.
° Trabajar con clases y objetos Number, String y Math: Aplicar funcionalidades avanzadas a números y cadenas, y llevar a cabo operaciones matemáticas complejas.
° Administrar arreglos: Almacenar y manipular datos en arreglos, ya sean unidimensionales o multidimensionales.
° Utilizar objetos Map: Crear y gestionar objetos Map para almacenar pares clave-valor y acceder eficazmente a los datos.

Resultado

Al adentrarse en el estudio y la implementación de estos conceptos, los profesionales dedicados al desarrollo Front-End adquieren una destreza sustancial en Javascript, lo que les capacita para la creación de aplicaciones web altamente interactivas y eficaces. Esto implica la habilidad de manipular información de diversa naturaleza, tener control absoluto sobre el flujo de ejecución de un programa, diseñar interfaces de usuario dinámicas y aprovechar al máximo las características avanzadas de este lenguaje de programación. La comprensión profunda de conceptos como clases, estructuras de datos y funciones avanzadas les brinda a los desarrolladores la capacidad de erigir aplicaciones web que son tanto robustas como altamente funcionales en su desempeño.

Semana 5:

Temas:

  • Desarrollo FrontEnd con Framework JS

Definiciones

1. Framework JS (JavaScript): Un conjunto de herramientas y bibliotecas que simplifican el desarrollo de aplicaciones web FrontEnd, proporcionando una estructura y funcionalidades predefinidas para facilitar la creación de interfaces de usuario dinámicas y funcionales.
2. Client Side Rendering (CSR): El proceso en el cual el renderizado de una página web ocurre en el navegador del cliente, utilizando JavaScript para generar y modificar dinámicamente el contenido de la página. Esto mejora la velocidad de carga y la interactividad de las aplicaciones web.
3. Gestión de Dependencias: La administración de bibliotecas y módulos necesarios para el desarrollo de una aplicación, a menudo llevada a cabo mediante herramientas como npm o Yarn, que permiten instalar, actualizar y gestionar las dependencias de un proyecto.
4. Desarrollo Basado en Componentes: Un enfoque de desarrollo que descompone la interfaz de usuario en componentes reutilizables e independientes. En este enfoque, se utilizan conceptos como JSX para describir la estructura del componente, PROPS para pasar datos entre componentes y CHILDREN para representar el contenido anidado dentro de un componente.
5. Estilos en Framework JS: La aplicación de estilos visuales a elementos de la interfaz de usuario en aplicaciones web FrontEnd desarrolladas con Frameworks JS, lo que incluye la gestión de la apariencia y el diseño de los componentes.
6. Styles Inline: La aplicación de estilos directamente en el elemento HTML utilizando atributos de estilo en línea, lo que permite definir estilos específicos para un elemento individual.
7. Style Sheets: El uso de hojas de estilo (CSS) para definir estilos visuales en una aplicación web, lo que permite la separación de la estructura y el diseño de una página web.
8. Styles Modules: Un enfoque que encapsula estilos específicos de un componente para evitar conflictos con otros estilos en la aplicación, proporcionando una solución de alcance limitado y aislado para los estilos de un componente.
9. Styles Components: La aplicación de estilos específicos de un componente utilizando bibliotecas de estilos proporcionadas por Frameworks JS, lo que permite un enfoque más modular y eficiente para la gestión de estilos en aplicaciones web.

Procedimiento

° Investigue la documentación del marco de JavaScript para comprender su estructura y funcionamiento.
° Emplee herramientas de gestión de paquetes como npm o Yarn para controlar las bibliotecas y módulos en su proyecto.
° Adopte el desarrollo orientado a componentes mediante el uso de JSX, PROPS y CHILDREN para diseñar componentes reutilizables.
° Incorpore estilos en su aplicación, seleccionando entre estilos en línea, hojas de estilo, módulos de estilos o estilos de componentes según sea necesario.
° Participe en el Laboratorio 05A para aprender cómo transmitir datos mediante props y children y presentarlos en un componente secundario.

Resultado

° Desarrolle de manera eficaz aplicaciones FrontEnd empleando un Framework de JavaScript.
° Comprenda minuciosamente la estructura y el funcionamiento del framework, lo que le brindará la capacidad de crear aplicaciones de cliente de manera altamente eficaz.
° Domine la gestión de las dependencias y utilice herramientas de creación de proyectos que simplificarán la configuración inicial de manera considerable.
° Cree interfaces de usuario dinámicas y reutilizables mediante el desarrollo orientado a componentes, aprovechando JSX, PROPS y CHILDREN.
° Aplique diversas técnicas de diseño, abarcando desde estilos en línea hasta la utilización de componentes de estilo y marcos CSS para moldear la apariencia de su aplicación.
° Lleve a cabo experimentos con la transferencia de datos mediante props y children en un componente secundario durante el Laboratorio 05A.

Temas:

  • Enfoque en el uso de Hooks en React.
  • Exploración de los Hooks predefinidos, como `useState`, `useEffect`, `useContext`, `useRef`, `useReducer`, `useCallback`, y `useMemo`
  • Creación y aplicación de Hooks personalizados
  • valuación de habilidades en el Laboratorio 09 como parte de la Evaluación de Logro 01

Definiciones

1. Hooks en React: Son funciones que permiten a los componentes funcionales de React tener estado y efectos secundarios. Son una característica fundamental para la gestión de la lógica en componentes funcionales.
2. useState Un hook de React que permite gestionar estados locales en componentes funcionales. Permite a los componentes funcionales mantener y actualizar su estado interno.
3. useEffect: Un hook de React que maneja efectos secundarios en componentes funcionales. Se utiliza para realizar tareas como la suscripción a datos externos o la manipulación del DOM después de que el componente se haya renderizado.
4. useContext: Un hook que permite acceder al contexto global de la aplicación en componentes de React. Facilita la transmisión de datos y estado a través de múltiples componentes sin la necesidad de pasar props manualmente.
5. useRef: Un hook que permite interactuar con elementos del DOM directamente. Se utiliza para acceder a nodos del DOM y mantener referencias a elementos dentro de componentes de React.
6. useReducer: Un hook que maneja estados complejos con una lógica más estructurada. Se asemeja a Redux y se utiliza cuando el estado de un componente es más complejo y requiere una lógica más detallada para actualizarse.
7. useCallback: Un hook que optimiza el rendimiento al memoizar funciones. Se utiliza para evitar la recreación innecesaria de funciones en componentes, lo que puede mejorar el rendimiento en aplicaciones de React.
8. useMemo: Un hook para optimizar cálculos costosos en componentes funcionales. Memoiza el resultado de una función costosa para evitar cálculos repetidos cuando las dependencias no han cambiado.
